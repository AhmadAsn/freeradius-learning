# 5. 802.1X Deployment Guide

This guide covers deploying 802.1X network access control using this FreeRADIUS stack — from switch configuration to supplicant setup, with VLAN assignment.

---

## What is 802.1X?

802.1X is a port-based network access control standard. It prevents devices from accessing the network until they authenticate:

```
┌──────────┐           ┌──────────┐           ┌──────────────┐
│  Client   │           │  Switch  │           │  FreeRADIUS   │
│(Supplicant)│◀── 802.1X ──▶│  (NAS)   │◀── RADIUS ──▶│   Server     │
└──────────┘           └──────────┘           └──────────────┘
     │                       │                        │
     │                  Port blocked                   │
     │── EAP Identity ────▶│── Access-Request ──────▶│
     │                      │                         │── Check user
     │◀── EAP Challenge ───│◀── Access-Challenge ───│
     │── EAP Response ────▶│── Access-Request ──────▶│
     │                      │                         │── Verify
     │◀── EAP Success ────│◀── Access-Accept ───────│
     │                      │      (VLAN 100)         │
     │                  Port opened                   │
     │                  VLAN assigned                  │
```

## Prerequisites

Before deploying 802.1X:

1. FreeRADIUS stack is running and healthy (`make status`)
2. Test auth works: `make test`
3. TLS certificates are generated (happens automatically on first boot)
4. You have the `ca.pem` file to distribute to clients

## Step 1: Configure NAS Devices

### Add your switch/AP to `clients.conf`

Every network device that sends RADIUS requests needs an entry:

```
# freeradius/config/clients.conf

client core-switch-01 {
    ipaddr    = 10.0.1.1
    secret    = MyStr0ngR@diusK3y!
    nastype   = cisco
    shortname = sw01
    require_message_authenticator = yes
}

client wireless-controller {
    ipaddr    = 10.0.2.1
    secret    = W1f1R@diusS3cret!
    nastype   = other
    shortname = wlc01
    require_message_authenticator = yes
}
```

Restart after adding clients:

```bash
make restart
```

### Cisco IOS switch configuration

```
! Enable AAA
aaa new-model
aaa authentication dot1x default group radius
aaa authorization network default group radius
aaa accounting dot1x default start-stop group radius

! RADIUS server
radius server FREERADIUS
 address ipv4 <FREERADIUS_HOST_IP> auth-port 1812 acct-port 1813
 key MyStr0ngR@diusK3y!

! Enable 802.1X globally
dot1x system-auth-control

! Configure access port
interface GigabitEthernet0/1
 switchport mode access
 switchport access vlan 999          ! Default "quarantine" VLAN
 authentication port-control auto
 authentication host-mode multi-auth  ! Multiple devices per port
 dot1x pae authenticator
 spanning-tree portfast
```

### Cisco WLC (Wireless LAN Controller)

1. **Security → AAA → RADIUS → Authentication**
   - Add server: IP = `<FREERADIUS_IP>`, Port = `1812`, Secret = `<shared_secret>`
2. **Security → AAA → RADIUS → Accounting**
   - Add server: same IP, Port = `1813`, same secret
3. **WLANs → Edit WLAN → Security → Layer 2**
   - Security: **WPA + WPA2**
   - Auth Key Mgmt: **802.1X**
4. **WLANs → Edit WLAN → Security → AAA Servers**
   - Select the RADIUS server

### Juniper EX switch

```
set access radius-server <FREERADIUS_IP> secret "MyStr0ngR@diusK3y!"
set access radius-server <FREERADIUS_IP> port 1812
set access radius-server <FREERADIUS_IP> accounting-port 1813

set protocols dot1x authenticator interface ge-0/0/1.0
set protocols dot1x authenticator interface ge-0/0/1.0 supplicant multiple
```

### Aruba / HPE switch

```
radius-server host <FREERADIUS_IP> key "MyStr0ngR@diusK3y!"
aaa port-access authenticator gi1/0/1
aaa port-access authenticator gi1/0/1 client-limit 10
aaa port-access authenticator active
```

## Step 2: Create Users and Groups

### Pre-configured groups

This stack ships with VLAN-assigning groups:

| Group | VLAN | Extra Attributes |
|-------|------|-----------------|
| `employees` | 100 | — |
| `guests` | 200 | 5 Mbps down / 2 Mbps up, max 1 session |
| `admins` | 10 | Cisco priv-lvl=15 |
| `contractors` | 150 | 8-hour session timeout |

### Add users via Makefile

```bash
# Add to employees group (VLAN 100)
make add-user USER=jdoe PASS='MyP@ssw0rd!' GROUP=employees

# Add to admins group (VLAN 10)
make add-user USER=netadmin PASS='Adm1nP@ss!' GROUP=admins

# Add to guests group (VLAN 200, bandwidth limited)
make add-user USER=visitor01 PASS='Gu3stP@ss!' GROUP=guests
```

### Add users via SQL

```bash
make db-shell
```

```sql
-- Add user credentials
INSERT INTO radcheck (username, attribute, op, value)
VALUES ('jdoe', 'Cleartext-Password', ':=', 'MyP@ssw0rd!');

-- Assign to group
INSERT INTO radusergroup (username, groupname, priority)
VALUES ('jdoe', 'employees', 1);
```

### Create a custom VLAN group

```sql
-- Create IoT VLAN group
INSERT INTO radgroupreply (groupname, attribute, op, value) VALUES
('iot-devices', 'Tunnel-Type',             ':=', 'VLAN'),
('iot-devices', 'Tunnel-Medium-Type',      ':=', 'IEEE-802'),
('iot-devices', 'Tunnel-Private-Group-Id', ':=', '300');
```

## Step 3: Distribute CA Certificate

Clients must trust the RADIUS server's CA certificate to establish the EAP-TLS tunnel. Without this, clients will either refuse to connect or prompt the user to trust an unknown certificate.

### Get the CA certificate

```bash
# Copy from the container
docker cp freeradius:/etc/raddb/certs/ca.pem ./ca.pem

# Or from the host mount
cat freeradius/config/certs/ca.pem
```

### Windows (GPO deployment)

1. Copy `ca.pem` to a network share
2. Open **Group Policy Management → Computer Configuration → Policies → Windows Settings → Security Settings → Public Key Policies → Trusted Root CAs**
3. Import `ca.pem`

### Windows (manual)

1. Rename `ca.pem` to `ca.crt`
2. Double-click → **Install Certificate → Local Machine → Trusted Root Certification Authorities**

### macOS (MDM profile)

Create a `.mobileconfig` profile:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
 "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>PayloadContent</key>
    <array>
        <dict>
            <key>PayloadCertificateFileName</key>
            <string>ca.pem</string>
            <key>PayloadContent</key>
            <data><!-- Base64 encoded CA cert --></data>
            <key>PayloadType</key>
            <string>com.apple.security.root</string>
        </dict>
    </array>
</dict>
</plist>
```

### Linux

```bash
sudo cp ca.pem /usr/local/share/ca-certificates/freeradius-ca.crt
sudo update-ca-certificates
```

## Step 4: Configure Supplicants

### Windows 10/11 (Wired 802.1X)

1. **Services** → Start **Wired AutoConfig** service (set to Automatic)
2. **Network adapter → Properties → Authentication tab**
3. Enable **IEEE 802.1X authentication**
4. Method: **Microsoft: Protected EAP (PEAP)**
5. **Settings:**
   - ✅ Validate server certificate
   - Select your CA in the trusted list
   - Auth method: **Secured password (EAP-MSCHAPv2)**

### Windows 10/11 (Wireless)

1. Connect to the SSID
2. When prompted, select **PEAP**
3. Enter username and password
4. If asked to trust the certificate, verify the CA name matches

### macOS

1. **System Preferences → Network → Wi-Fi → Advanced**
2. Add the SSID
3. Security: **WPA2 Enterprise**
4. Mode: **PEAP**
5. Enter username and password
6. When prompted, trust the certificate

### Linux (NetworkManager)

```bash
nmcli connection add type wifi \
    con-name "Corporate" \
    ifname wlan0 \
    ssid "CorpWiFi" \
    wifi-sec.key-mgmt wpa-eap \
    802-1x.eap peap \
    802-1x.identity "jdoe" \
    802-1x.password "MyP@ssw0rd!" \
    802-1x.phase2-auth mschapv2 \
    802-1x.ca-cert "/path/to/ca.pem"
```

### Linux (wpa_supplicant)

```
# /etc/wpa_supplicant/wpa_supplicant.conf
network={
    ssid="CorpWiFi"
    key_mgmt=WPA-EAP
    eap=PEAP
    identity="jdoe"
    password="MyP@ssw0rd!"
    phase2="auth=MSCHAPV2"
    ca_cert="/path/to/ca.pem"
}
```

### Android

1. **Settings → Wi-Fi → Add network**
2. SSID: Your network name
3. Security: **WPA/WPA2 Enterprise**
4. EAP method: **PEAP**
5. Phase 2: **MSCHAPV2**
6. CA certificate: Import `ca.pem` (or "Do not validate" for testing only)
7. Identity: username
8. Password: password

### iOS / iPadOS

Deploy via MDM configuration profile for best results. Manual:

1. **Settings → Wi-Fi → select SSID**
2. Enter username and password
3. Trust the certificate when prompted

## Step 5: MAC Authentication Bypass (MAB)

For devices that can't do 802.1X (printers, IP phones, IoT), use MAB:

### How MAB works

The switch sends the device's MAC address as both the username and password.

### Configure in FreeRADIUS

The `canonicalization` policy normalizes MAC addresses to `aa-bb-cc-dd-ee-ff` format.

Add the device to the database:

```sql
-- MAC = AA:BB:CC:DD:EE:FF → stored as aa-bb-cc-dd-ee-ff
INSERT INTO radcheck (username, attribute, op, value)
VALUES ('aa-bb-cc-dd-ee-ff', 'Cleartext-Password', ':=', 'aa-bb-cc-dd-ee-ff');

INSERT INTO radusergroup (username, groupname, priority)
VALUES ('aa-bb-cc-dd-ee-ff', 'iot-devices', 1);
```

### Configure the switch (Cisco IOS)

```
interface GigabitEthernet0/2
 switchport mode access
 authentication port-control auto
 authentication host-mode single-host
 dot1x pae authenticator
 mab                                  ! Enable MAB
 authentication order dot1x mab       ! Try 802.1X first, fall back to MAB
 authentication priority dot1x mab
 spanning-tree portfast
```

## Step 6: Verify and Monitor

### Test authentication

```bash
# Quick test
make test

# All pre-configured users
make test-all

# Specific user
docker exec freeradius radtest jdoe 'MyP@ssw0rd!' 127.0.0.1 0 testing123
```

### Check logs

```bash
# Real-time FreeRADIUS logs
make logs-radius

# Look for successful auth
docker exec freeradius grep "Login OK" /var/log/freeradius/radius.log

# Look for failures
docker exec freeradius grep "Login incorrect" /var/log/freeradius/radius.log
```

### View accounting data

```bash
make db-shell
```

```sql
-- Active sessions
SELECT username, nasipaddress, acctstarttime, framedipaddress
FROM radacct
WHERE acctstoptime IS NULL;

-- Recent authentications
SELECT username, reply, authdate
FROM radpostauth
ORDER BY id DESC
LIMIT 20;
```

### Debug mode

If something isn't working:

```bash
make debug
```

This restarts FreeRADIUS in verbose mode. Look for:
- `Found Auth-Type = ...` — confirms credential lookup
- `Login OK` / `Login incorrect` — auth result
- `Tunnel-Private-Group-Id` — VLAN assignment
- TLS handshake errors — certificate issues

## VLAN Assignment Flow

```
 User authenticates
         │
         ▼
 FreeRADIUS checks radcheck ──▶ Credentials valid?
         │                              │
         │ Yes                          │ No
         ▼                              ▼
 Look up radusergroup ──────▶    Access-Reject
         │
         ▼
 Load radgroupreply attrs
 (Tunnel-Type, Tunnel-Private-Group-Id)
         │
         ▼
 Access-Accept + VLAN attributes
         │
         ▼
 Switch assigns port to VLAN
```

### Example: what the switch receives

```
Received Access-Accept
    Tunnel-Type:0 = VLAN
    Tunnel-Medium-Type:0 = IEEE-802
    Tunnel-Private-Group-Id:0 = "100"
```

The switch moves the port from the default VLAN to VLAN 100.
